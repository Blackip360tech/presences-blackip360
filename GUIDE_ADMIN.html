1. Introduction

BlackIP360 Présences est un système de gestion des présences qui s'intègre avec Microsoft 365 (Azure AD + SharePoint). L'application permet aux employés de pointer leur statut en temps réel, de soumettre des demandes de congé, et aux administrateurs de générer des rapports de paie.

Technologies : HTML/CSS/JavaScript (pas de backend), MSAL.js pour l'authentification, Microsoft Graph API pour la lecture/écriture SharePoint.

2. Configuration actuelle

Tous les paramètres sont dans le fichier config.js :

  • CLIENT_ID — ID de l'application Azure AD
  • TENANT_ID — ID du tenant Microsoft 365
  • SHAREPOINT_HOST — domaine SharePoint
  • SHAREPOINT_SITE_PATH — chemin du site SharePoint
  • SHAREPOINT_LIST — nom de la liste des pointages
  • SHAREPOINT_LIST_SOLDES — liste des soldes de congés
  • SHAREPOINT_LIST_DEMANDES — liste des demandes de congé
  • APP_URL — URL publique GitHub Pages
  • STATUTS — liste des 11 statuts disponibles
  • DEPARTEMENTS — liste des départements
  • TYPES_CONGE — types de congé pour les demandes

3. Ajouter un utilisateur

Aucune action requise. Tout employé avec un compte @blackip360.com dans Azure AD peut se connecter automatiquement à l'application.

Le premier pointage crée l'entrée SharePoint de l'employé. Ensuite l'admin peut :

  • Lui assigner un département via l'onglet Accès
  • Initialiser ses soldes de vacances et maladie

4. Donner l'accès administrateur

Deux façons de rendre un utilisateur administrateur :

Par email (immédiat)

Modifier la liste adminEmails dans app.js (méthode _checkAdmin()) :

const adminEmails = [
  'admin@blackip360.com',
  'tech@blackip360.com',
  'tfournier@blackip360.com',
  // Ajouter ici
];

Par département Azure AD

Toute personne dont le département dans Azure AD est Direction devient automatiquement admin. À configurer depuis le portail Azure → Utilisateurs → Profil → Département.

5. Modifier les statuts

Les 11 statuts sont définis dans config.js, tableau STATUTS. Chaque statut a :

PropriétéDescriptionExemple
idIdentifiant uniquebureau
labelTexte affiché (enregistré dans SharePoint)Je suis là au bureau
iconEmoji🏢
colorCouleur hex du bouton#198754
categorypresent ou absentpresent
⚠️ Le label est la valeur stockée en SharePoint. Ne pas le modifier sans mettre à jour les données existantes — sinon les pointages antérieurs perdent leur icône/couleur.

Note : les statuts avec id: route_bip ou route_cv247 forcent une note obligatoire au moment du pointage (nom du client).

6. Modifier les départements

Dans config.js, tableau DEPARTEMENTS :

DEPARTEMENTS: [
  'Tous',
  'Direction',
  'Développement',
  'Infrastructure',
  'Support',
  'Administration',
],

Le premier élément doit toujours être 'Tous' (utilisé dans les filtres). Les autres sont modifiables librement.

L'assignation du département à un employé se fait depuis l'onglet Accès de l'application.

7. Gestion des congés et soldes

Types de congé

  • 🌴 Vacances — déduit du solde vacances quand approuvé
  • 🤒 Maladie — déduit du solde maladie quand approuvé
  • 📋 Sans solde — enregistré sans déduction

Workflow de demande

  1. L'employé soumet une demande dans l'onglet Mes demandes
  2. La demande apparaît chez l'admin (même onglet, section admin)
  3. L'admin clique Approuver ou Refuser avec une note optionnelle
  4. Si approuvée pour Vacances/Maladie, le solde de l'employé est automatiquement déduit

Initialiser les soldes

Pour chaque employé, aller dans l'onglet Accès → Gestion des utilisateurs et entrer :

  • Vacances (h) — ex: 80 pour 10 jours
  • Maladie (h) — ex: 40 pour 5 jours

9. Déploiement

L'application est hébergée sur GitHub Pages via GitHub Actions. Tout commit sur la branche main déclenche un déploiement automatique.

  • Environnement DEV : https://blackip360tech.github.io/presences-blackip360-dev
  • Environnement PROD : https://blackip360tech.github.io/presences-blackip360

Après un commit, le déploiement prend environ 2 minutes. Vérifier l'état dans l'onglet Actions du dépôt GitHub.
